# Knowledge Graph Builder - 知识图谱构建器

## 概述

本技能是微课视频生产线的核心内容构建技能，负责将课程大纲、教学材料和用户描述自动构建为结构化知识图谱。通过知识点提取、关系识别、难度评估和学习路径规划，为微课的模块划分和脚本生成提供坚实的知识结构基础。

## 核心功能

### 1. 知识点提取

**提取模式：**

| 输入类型 | 提取策略 | 示例 |
|---------|---------|------|
| 课程大纲 | 结构化解析 | 识别章节标题→知识点 |
| 教学文档 | NLP实体识别 | 识别术语、概念、原理 |
| 视频/音频 | 语音转文本+提取 | 提取关键内容点 |
| 自然语言描述 | 意图识别+结构化 | "需要讲清楚XXX" |

### 2. 关系抽取

**关系类型体系：**

| 关系类型 | 符号 | 说明 | 逆关系 |
|---------|------|------|-------|
| 包含关系 | ⊃ | 上位概念包含下位概念 | ⊂ |
| 并列关系 | ∥ | 同级别概念 | ∥ |
| 递进关系 | → | 前后顺序、难度递进 | ← |
| 因果关系 | ⇒ | 前因后果 | ⇐ |
| 对比关系 | ⇔ | 异同比较 | ⇔ |
| 应用关系 | ↱ | 理论→实践应用 | ↰ |
| 组成关系 | ⊕ | 部分→整体 | ⊖ |
| 发展阶段 | ⟶ | 时间/成熟顺序 | ⟵ |

**关系识别规则：**


### 3. 难度评估

**难度评估维度：**

| 维度 | 评估因素 | 权重 |
|------|---------|------|
| 抽象程度 | 概念的具体/抽象程度 | 0.25 |
| 复杂性 | 涉及的概念数量和关系复杂度 | 0.25 |
| 前置依赖 | 需要的前置知识深度 | 0.20 |
| 专业术语密度 | 专业术语的数量和罕见度 | 0.15 |
| 计算/推理要求 | 是否需要计算或复杂推理 | 0.15 |

### 4. 学习路径规划

**路径规划算法：**

```python
学习路径规划 = 函数(
    知识图谱,
    学习者画像,
    学习目标
) -> 学习路径列表

规划原则:
  1. 拓扑排序: 确保前置知识先于依赖知识
  2. 难度递进: 相邻节点难度差≤1
  3. 模块内聚: 相关知识点尽量在同一模块内
  4. 路径最短: 减少不必要的前置依赖
  5. 个性化适配: 根据学习者水平调整路径
```

**路径可视化表示：**

```
学习路径示例:
┌─────────────────────────────────────────────────────────────┐
│                    课程: 发展心理学                          │
├─────────────────────────────────────────────────────────────┤

│   KN_001(1) → KN_002(1) → KN_003(2)                        │
│                                                             │
│ 模块2: 婴儿期发展                                           │
│   KN_004(2) → KN_005(2) → KN_006(3)                        │

│   [需要: KN_001]                                            │

│        ↑                                                    │
│   [需要: KN_001, KN_004]                                    │
└─────────────────────────────────────────────────────────────┘
```

### 5. 模块划分

**模块划分策略：**

| 策略 | 适用场景 | 划分依据 |
|------|---------|---------|
| 主题聚合 | 内容有明确主题 | 主题相关性 |
| 难度分级 | 内容有明显难度差异 | 难度等级 |
| 时间分段 | 需要控制模块时长 | 学习时段 |
| 独立性优先 | 需要灵活组合 | 依赖关系最小化 |

## 最佳实践

### 1. 输入优化

**高质量输入示例：**
```
课程大纲结构:
第一章 发展心理学概述
  1.1 发展心理学的定义与研究对象
  1.2 发展心理学的基本原则
  1.3 发展的主要理论流派

第二章 婴儿期发展（0-3岁）
  2.1 感知觉的发展
  2.2 动作与运动发展
  2.3 言语与沟通发展
```

**简化输入处理：**
```
用户: "我想讲讲儿童成长的几个关键阶段"
系统自动推断:
  - 识别潜在知识点: 婴儿期、幼儿期、儿童期、青春期
  - 识别潜在关系: 递进关系（按年龄）
  - 识别主题: 儿童发展
```

### 2. 图谱质量保障

**质量检查清单：**
- [ ] 无孤立节点（除初始节点外）
- [ ] 无循环依赖
- [ ] 难度递增平滑
- [ ] 模块时长均衡
- [ ] 知识点命名一致

### 3. 常见问题处理

| 问题 | 处理策略 |
|------|---------|
| 知识点边界模糊 | 拆分或合并，基于教学粒度 |
| 关系类型不确定 | 标注为"关联"关系，后续人工确认 |
| 循环依赖 | 打断循环，保留最强关系 |
| 难度评估偏差 | 人工校准+反馈调整 |

## 故障排除

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 知识点遗漏 | 输入信息不完整 | 补充输入或人工补充 |
| 关系识别错误 | 歧义句式干扰 | 增加上下文信息或人工纠正 |
| 难度评估不准 | 缺乏领域知识 | 人工校准+领域规则 |
| 模块划分不合理 | 划分策略不当 | 调整划分策略或手动调整 |

## 技术实现框架

```python
class KnowledgeGraphBuilder:
    def __init__(self):
        self.entity_extractor = EntityExtractor()
        self.relation_extractor = RelationExtractor()
        self.difficulty_assessor = DifficultyAssessor()
        self.path_planner = PathPlanner()
        self.module_divider = ModuleDivider()
        self.graph_optimizer = GraphOptimizer()
    
    def build(
        self,
        source_data: dict,
        options: dict = None
    ) -> KnowledgeGraph:
        # 1. 知识点提取
        nodes = self.entity_extractor.extract(source_data)
        
        # 2. 关系抽取
        edges = self.relation_extractor.extract(source_data, nodes)
        
        # 3. 图谱构建
        graph = self.construct_graph(nodes, edges)
        
        # 4. 难度评估
        graph = self.difficulty_assessor.assess(graph)
        
        # 5. 路径规划
        paths = self.path_planner.plan(graph)
        
        # 6. 模块划分
        modules = self.module_divider.divide(graph, paths)
        
        # 7. 优化验证
        graph = self.graph_optimizer.optimize(graph, modules)
        
        return KnowledgeGraph(
            nodes=graph.nodes,
            edges=graph.edges,
            modules=modules,
            paths=paths
        )
```
